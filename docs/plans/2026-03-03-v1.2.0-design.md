# v1.2.0 Pipeline Robustness — Design

> **Module:** `github.com/anatolykoptev/go-engine` | **Depends on:** go-stealth upstream changes

**Goal:** Make the pipeline more robust with fetch+chunk fusion, smart chunking, token budget awareness, and production-grade retry infrastructure.

**Scope:** Two repos — go-stealth (retry primitives) and go-engine (pipeline + integration).

**Sources:** [Firecrawl](https://github.com/firecrawl/firecrawl) (retryTracker, trimToTokenLimit), [GoClaw](https://github.com/nextlevelbuilder/goclaw) (RetryHookFunc, retrySend with resetFn)

---

## 1. Fetch + Immediate Chunk

**Package:** `pipeline/fetch.go`

**Current:** `ParallelFetch` returns `map[string]string` (url → extracted text). Then `pipeline/run.go` does a separate chunk+filter pass over all results.

**Change:** The wrapping function inside each goroutine calls `fetch → extract → chunk → join` in a single pass. The separate `chunkAndFilter` step in `run.go` is removed.

**Return type unchanged:** `map[string]string` — content is already chunked+filtered when returned.

**Benefit:** Reduces latency (chunk while other fetches are in flight), simplifies `Run()`.

**LOC:** ~30 changed in pipeline/fetch.go, ~10 removed from pipeline/run.go.

---

## 2. Auto-Chunking Detection

**Package:** `text/chunker.go`

**Current:** `CharacterChunker.Chunk()` already returns a single-element slice when text ≤ chunkSize (line 26-28). However, it still allocates `[]rune` and processes the text.

**Change:** Add `NeedsChunking(text string) bool` method to `CharacterChunker`. Returns `true` if `utf8.RuneCountInString(text) > c.chunkSize`. The merged fetch goroutine (item 1) calls `NeedsChunking` before `Chunk` to skip processing entirely for short content.

**LOC:** ~15 in text/chunker.go.

## 3. Token Budget Truncation

**Package:** `text/token.go` (new file)

```go
const DefaultCharsPerToken = 3.5

func EstimateTokens(text string, charsPerToken float64) int
func TruncateToTokenBudget(text string, maxTokens int, charsPerToken float64) string
```

**Current:** `pipeline/run.go` passes hardcoded `contentLimitChars=6000` to `llm.Summarize`. `llm/summarize.go:BuildSourcesText` truncates per-source content to this char limit.

**Change:**
- `text/token.go` provides estimation and truncation.
- `llm/summarize.go` accepts `maxTokens int` instead of `contentLimit int`. Uses `TruncateToTokenBudget` to trim content per source.
- `Pipeline` config gets `WithMaxTokens(n int)` option (default: 4096).
- Default chars/token: 3.5 (English). Configurable via `WithCharsPerToken(f float64)` for multilingual workloads (~2.5).

**Approach:** Character estimation (Firecrawl fallback pattern: 2.8 chars/token). Zero dependencies. Accurate within ~15% which is sufficient since we apply an 80% safety margin.

**LOC:** ~50 in text/token.go, ~20 in llm/summarize.go, ~10 in pipeline/pipeline.go.

## 4. Retry Hooks

**Package:** `go-stealth/retry.go` (upstream)

```go
type RetryHookFunc func(ctx context.Context, attempt int, maxAttempts int, err error)

func WithRetryHook(ctx context.Context, fn RetryHookFunc) context.Context
func retryHookFromContext(ctx context.Context) RetryHookFunc
```

**Pattern:** GoClaw's context-injected hook. `RetryHTTP` and `RetryDo` extract hook from context, call before each attempt. nil hook = noop.

**go-engine integration:** `Pipeline` config gets `WithRetryHook(fn)` option. Passes hook-enriched context to `fetch.Fetcher.FetchBody()`.

**LOC:** ~20 in go-stealth, ~10 in go-engine fetch/ + pipeline/.

## 5. Retry with Reset

**Package:** `go-stealth/retry.go` (upstream)

Add `ResetFn func()` optional parameter to `RetryHTTP` and `RetryDo`. Called before each retry (not before the first attempt). nil = no reset.

**Option A:** New functions `RetryHTTPWithReset`, `RetryDoWithReset` (preserves backward compat).

**Option B:** Add `ResetFn func()` field to `RetryConfig` struct.

**Chosen: Option A** — separate functions, cleaner API, no breaking changes.

**Use case in go-engine:** Reset request body readers (e.g., `bytes.NewReader.Seek(0, 0)`) before retrying POST requests.

**LOC:** ~15 in go-stealth.

## 6. Per-URL Retry Tracker

**Package:** `go-stealth/retry_tracker.go` (new file, upstream)

```go
type RetryTracker struct {
    mu          sync.Mutex
    entries     map[string]*retryEntry
    ttl         time.Duration
    maxAttempts int
}

type retryEntry struct {
    Attempts    int
    LastErr     error
    LastAttempt time.Time
    Permanent   bool
}

func NewRetryTracker(maxAttempts int, ttl time.Duration) *RetryTracker
func (t *RetryTracker) ShouldRetry(url string) bool
func (t *RetryTracker) RecordAttempt(url string, err error)
func (t *RetryTracker) RecordSuccess(url string)
```

**Behavior:**
- `ShouldRetry` returns false if Permanent or Attempts >= maxAttempts.
- `RecordAttempt` increments counter, sets Permanent if error is permanent (e.g., 404, 410).
- TTL cleanup: entries older than ttl are evicted on `ShouldRetry` calls (lazy eviction).
- Thread-safe via sync.Mutex.

**go-engine integration:** `fetch.Fetcher` gets optional `WithRetryTracker(t)` option. `FetchBody` checks `ShouldRetry(url)` before request, calls `RecordAttempt`/`RecordSuccess` after. Tracker persists across `Pipeline.Run()` calls.

**LOC:** ~50 in go-stealth, ~15 in go-engine fetch/.
