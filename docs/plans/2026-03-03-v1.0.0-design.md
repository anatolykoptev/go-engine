# go-engine v1.0.0 Design

> **Date:** 2026-03-03 | **Status:** Approved

## Goal

API freeze + architecture upgrade. Transform go-engine from a collection of packages into a composable framework with swappable strategies and a generic source adapter pattern.

**Inspiration:** Haystack (Component interface) + Crawl4ai (Strategy pattern) = Hybrid approach.

---

## 1. `sources/` — Source Interface + API Client

### Problem

`sources/` is a stub (doc.go only). go-search has 6 hardcoded adapters in `internal/engine/sources/`. No shared interface — each consumer reimplements API calls.

### Design

**`sources/source.go`** — interface + types (~60 LOC)

```go
type Source interface {
    Name() string
    Search(ctx context.Context, q Query) ([]Result, error)
}

type Query struct {
    Text      string
    Limit     int
    TimeRange string            // "day", "week", "month", "year", ""
    Language  string            // ISO 639-1
    Extra     map[string]string // source-specific params
}

type Result struct {
    Title    string
    URL      string
    Content  string
    Score    float64
    Metadata map[string]string
}
```

**`sources/apiclient.go`** — generic JSON API helper (~90 LOC)

```go
type APIClient struct {
    base       string
    httpClient *http.Client
    auth       AuthMethod
    rateLimit  *rate.Limiter
    metrics    *metrics.Registry
}

func NewAPIClient(base string, opts ...APIOption) *APIClient
func (c *APIClient) Get(ctx context.Context, path string, params url.Values, dest any) error
func (c *APIClient) Post(ctx context.Context, path string, body any, dest any) error
```

**`sources/auth.go`** — auth methods (~40 LOC)

```go
type AuthMethod interface {
    Apply(req *http.Request)
}

func BearerAuth(token string) AuthMethod
func NoAuthMethod() AuthMethod
```

### Consumer usage (go-search)

```go
type HackerNews struct{ api *sources.APIClient }

func (h *HackerNews) Name() string { return "hackernews" }
func (h *HackerNews) Search(ctx context.Context, q sources.Query) ([]sources.Result, error) {
    var resp hnResponse
    if err := h.api.Get(ctx, "/search", params, &resp); err != nil {
        return nil, err
    }
    return mapToResults(resp.Hits), nil
}
```

## 2. Strategy Interfaces + Implementations

### Problem

Extraction, chunking, and filtering are hardcoded. No way to swap strategies.

### Design

**`extract/strategy.go`** (~15 LOC)

```go
type Strategy interface {
    Extract(body []byte, pageURL *url.URL) (*Result, error)
}
// Existing Extractor already satisfies Strategy.
```

**`text/chunker.go`** — interface + CharacterChunker (~80 LOC)

```go
type Chunker interface {
    Chunk(text string) []string
}

type CharacterChunker struct {
    chunkSize int
    overlap   int
}

func NewCharacterChunker(chunkSize, overlap int) *CharacterChunker
```

CharacterChunker splits at word boundaries near `chunkSize`, with `overlap` characters shared between adjacent chunks. Respects UTF-8 rune boundaries.

**`text/filter.go`** — interface + BM25Filter (~140 LOC)

```go
type Filter interface {
    Filter(chunks []string, query string) []string
}

type BM25Filter struct {
    topK int
    k1   float64 // default: 1.5
    b    float64 // default: 0.75
}

func NewBM25Filter(topK int) *BM25Filter
```

BM25Filter tokenizes query and chunks, computes IDF across chunks, scores each chunk, returns top-K by score. Reference: Crawl4ai `content_filter_strategy.py`.

## 3. Pipeline Upgrade

### Problem

`ParallelFetch` has unbounded goroutines and silently skips errors. No orchestrator for the full pipeline.

### Design

**`pipeline/fetch.go`** — bounded ParallelFetch (~60 LOC)

```go
type FetchResult struct {
    URL     string
    Content string
    Err     error
}

func ParallelFetch(ctx context.Context, urls []string,
    fetchFn func(ctx context.Context, url string) (string, error),
    opts ...FetchOption) []FetchResult

func WithMaxConcurrency(n int) FetchOption
```

Breaking change: returns `[]FetchResult` instead of `map[string]string`. Errors are preserved, not skipped.

**`pipeline/run.go`** — orchestrator (~80 LOC)

```go
type Pipeline struct {
    sources   []sources.Source
    fetcher   *fetch.Fetcher
    extractor extract.Strategy
    chunker   text.Chunker
    filter    text.Filter
    llm       *llm.Client
}

func New(opts ...Option) *Pipeline
func (p *Pipeline) Run(ctx context.Context, query string) (*SearchOutput, error)
```

Pipeline flow: sources.Search → fetch URLs → extract content → chunk → filter → llm.Complete.

## 4. Singleflight in `cache/`

### Problem

Concurrent cache misses for the same key trigger N identical fetch operations (thundering herd).

### Design

Add `GetOrFetch` to `Tiered` using `golang.org/x/sync/singleflight` (~30 LOC):

```go
func (t *Tiered) GetOrFetch(ctx context.Context, key string, ttl time.Duration,
    fetchFn func(ctx context.Context) ([]byte, error)) ([]byte, error)
```

Only one goroutine executes `fetchFn` per key; concurrent callers share the result.
