# v1.3.0 Extraction Quality — Design

> **Module:** `github.com/anatolykoptev/go-engine` | **Current:** v1.2.0

**Goal:** Improve extraction output quality with enhanced cleanup, LLM fallback for thin content, format-aware extraction, and configurable system prompts.

**Scope:** 4 items. Deferred to v1.4.0: content-type aware preprocessing, per-strategy retry, LLM streaming.

**Sources:** [Firecrawl](https://github.com/firecrawl/firecrawl) (extractSmartScrape, removeUnwantedElements), [llm-scraper](https://github.com/mishushakov/llm-scraper) (cleanup.ts, preprocess format enum), [go-trafilatura](https://github.com/markusmobius/go-trafilatura) (docCleaning, postCleaning), [go-readability](https://github.com/go-shiori/go-readability) (prepDocument, prepArticle)

---

## Research Finding: Trafilatura Already Handles Tier 1 Cleanup

go-trafilatura performs 3-phase cleanup internally: `docCleaning` (strip 30+ tags), `convertTags` (normalize), `postCleaning` (attribute whitelist). Pre-cleanup before Tier 1 is redundant and potentially harmful (removes structural signals trafilatura needs). Tier 2/3 need enhancement instead.

---

## 1. Enhanced Tier 2/3 Cleanup

**Package:** `extract/goquery.go`, `extract/regex.go`, `extract/cleanup.go` (new)

**Current:** `goquery.go` strips 15 selectors (script, style, noscript, iframe, svg, header, footer, nav, aside, 4 class/role selectors). No attribute stripping. Regex tier strips all tags with no structured cleanup.

**Change:**

1. **Extend goquery selectors** — add ARIA roles, ad patterns, hidden elements:
   ```
   [aria-hidden=true], [hidden], .ad-*, #sidebar-*, .social-share,
   .cookie-banner, .popup, .modal, .newsletter-signup
   ```

2. **Attribute stripping** in goquery output — whitelist approach (trafilatura pattern):
   - Allow: `href`, `src`, `alt`, `title`, `lang`, `colspan`, `rowspan`
   - Strip everything else (style, class, id, data-*, aria-*, on*)

3. **Tokenizer fast-path** in `extract/cleanup.go` — for bodies > 500KB, use `html.NewTokenizer` to strip `<script>` and `<style>` blocks before DOM parse. Reduces memory for pathological pages.

**No pre-cleanup before Tier 1.** Trafilatura handles it internally.

**LOC:** ~40 in goquery.go/regex.go, ~15 in cleanup.go.

## 2. LLM-Backed Extraction Fallback

**Package:** `extract/extractor.go`, `extract/llm_fallback.go` (new)

**Current:** 3-tier chain (trafilatura → goquery → regex). If all produce thin content, pipeline gets garbage.

**Change:**

- **Threshold detection:** If `len(result.Content) < MinExtractChars` (default 200) after all 3 tiers, trigger LLM fallback.
- **Callback injection:**
  ```go
  type LLMFallbackFunc func(ctx context.Context, cleanedHTML string) (string, error)

  func WithLLMFallback(fn LLMFallbackFunc) Option
  func WithMinExtractChars(n int) Option
  ```
- **Flow:** Extractor.Extract → 3-tier chain → check threshold → if below, call fallbackFn with cleaned HTML (script/style already stripped) → use LLM output as content.
- **Extractor.Extract signature change:** Add `context.Context` parameter for LLM call:
  ```go
  func (e *Extractor) Extract(ctx context.Context, body []byte, pageURL *url.URL) (*Result, error)
  ```
  This is a breaking change to the `Strategy` interface. All callers updated.
- **nil fallback = no LLM call** (current behavior preserved, zero cost).
- **Pipeline wiring:** Pipeline passes `llm.CompleteWithSystem(ctx, extractionSystemPrompt, html)` as the fallback function.

**LOC:** ~50 in extract/, ~15 in pipeline/.

## 3. Format Enum on Extract Result

**Package:** `extract/strategy.go`, `extract/extractor.go`, all tiers

**Current:** `Result` has `Content` (text) and `Markdown` (sometimes filled by trafilatura, ignored by pipeline).

**Change:**

```go
type Format string

const (
    FormatText     Format = "text"      // plain text (default)
    FormatMarkdown Format = "markdown"  // markdown
    FormatHTML     Format = "html"      // cleaned HTML
)
```

- Add `WithFormat(f Format)` option on Extractor (default `FormatText` for backward compat).
- `Result.Content` always contains content in the requested format.
- `Result.Markdown` field **removed** — redundant with format enum.
- **Tier 1 (trafilatura):** Already produces both text and markdown. Returns requested format.
- **Tier 2 (goquery):**
  - `FormatText`: current behavior (cleaned text).
  - `FormatMarkdown`: pipe cleaned HTML through `htmltomarkdown.ConvertString()` (already a dependency).
  - `FormatHTML`: return cleaned HTML as-is.
- **Tier 3 (regex):** Always returns text regardless of format (best effort).
- Pipeline gets `WithExtractFormat(f)` option that configures the extractor.

**LOC:** ~60 across extract/ files, ~10 in pipeline/.

## 4. Configurable System Prompt

**Package:** `llm/client.go`

**Current:** `Complete(ctx, prompt)` and `CompleteParams(ctx, prompt, temperature, maxTokens)` — no system prompt parameter. `Summarize*` methods use hardcoded system prompts from `prompt.go`.

**Change:**

```go
func (c *Client) CompleteWithSystem(ctx context.Context, system, prompt string) (string, error)
```

- New method alongside existing `Complete` (backward compat preserved).
- Empty system string = no system message (matches current behavior).
- `Summarize*` methods unchanged — they already manage their own prompts.
- Used by: LLM fallback extraction (item 2), custom pipeline callers.

**LOC:** ~20 in llm/client.go.

## Breaking Changes

| Change | Impact | Migration |
|--------|--------|-----------|
| `Strategy.Extract` gains `ctx` parameter | All Strategy implementations + callers | Add `ctx context.Context` as first param |
| `Result.Markdown` removed | Anyone reading Markdown field | Use `Result.Content` with `FormatMarkdown` |

Both changes are in `extract/` package only. Pipeline callers (the main consumer) are updated in the same release.
