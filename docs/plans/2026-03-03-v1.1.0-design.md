# go-engine v1.1.0 — Search Quality Design

**Goal:** Improve search result quality through unified types, rate limit detection, result fusion, deduplication, and LLM-ready formatting.

**Decisions:**
- crypto/rand jitter removed from scope (lives in go-stealth)
- `search.Result` deleted, `sources.Result` used everywhere (breaking change)
- Fusion, dedup, markdown live in `search/` package
- Rate limit detection: auto-retry then typed error

---

## 1. Unified Result Type

Delete `search.Result`. Use `sources.Result{Title, URL, Content, Score, Metadata}` everywhere.

**Changes:**
- `search/result.go` — delete file; move `searxngResponse` to `searxng.go`
- `search/ddg.go`, `startpage.go`, `searxng.go` — return `[]sources.Result`
- `search/filter.go` — `FilterByScore`, `DedupByDomain` accept `[]sources.Result`
- `search/direct.go` — `SearchDirect` returns `[]sources.Result`
- `pipeline/convert.go` — remove `toSearchResults()` (no longer needed)

Engine-specific data goes in `Metadata`: `{"engine": "ddg"}`.

## 2. Rate Limit Detection

New typed error in `search/errors.go`:

```go
type ErrRateLimited struct {
    Engine     string
    RetryAfter time.Duration // 0 = unknown
}
func (e *ErrRateLimited) Error() string
```

**DDG detection** (`search/ddg.go`):
- Before parsing HTML: check for captcha markers ("Please try again", "robot", CAPTCHA form elements)
- Check HTTP 403/429
- On detection: one retry with increased backoff → if still captcha → return `ErrRateLimited`

**Startpage detection** (`search/startpage.go`):
- Check for "rate limited", "too many requests" in HTML body
- HTTP 429/403
- Same retry + typed error pattern

**SearchDirect** (`search/direct.go`):
- If one engine returns `ErrRateLimited` — others continue
- If ALL engines rate limited — propagate `ErrRateLimited` to caller

## 3. Per-Engine Rate Limiter

Each direct scraper gets its own `rate.Limiter` to prevent hammering.

```go
// search/direct.go
type DirectConfig struct {
    DDGLimiter       *rate.Limiter // default: 2 rps, burst 3
    StartpageLimiter *rate.Limiter // default: 1 rps, burst 2
}
```

- Called before each request to engine
- SearXNG already has limiter via `sources.APIClient` — no duplication
- Conservative defaults (DDG is sensitive to frequent requests)

## 4. WRR Result Fusion

Weighted Reciprocal Rank — merges results from multiple engines with configurable weights.

```go
// search/fusion.go
func FuseWRR(resultSets [][]sources.Result, weights []float64) []sources.Result
```

Algorithm:
1. For each result: `score += weight[i] / (k + rank)` where k=60 (standard RRF constant)
2. Group by URL — same URL from multiple engines accumulates score
3. Sort by score descending
4. Return merged list

Default weights: DDG=1.0, SearXNG=1.0, Startpage=0.8.

## 5. Snippet Deduplication

Bag-of-Words cosine similarity to remove near-duplicate snippets.

```go
// search/dedup.go
func DedupSnippets(results []sources.Result, threshold float64) []sources.Result
```

- Tokenize: lowercase + split on non-letter characters
- Build BoW vector per Content field
- Pairwise cosine similarity
- If similarity > threshold (default 0.85) — keep result with higher Score
- O(n²) but n < 30 typically — acceptable

## 6. SearXNG Categories

Pass `categories` parameter through `sources.Query.Extra`.

```go
q := sources.Query{
    Text:  "golang rate limiter",
    Extra: map[string]string{"categories": "it"},
}
```

SearXNG client reads `q.Extra["categories"]` and adds to query params. Default: `general`.

## 7. Markdown Output

```go
// search/format.go
func ResultsToMarkdown(results []sources.Result) string
```

Format:
```markdown
## 1. [Title](URL)
Snippet text here...

## 2. [Title](URL)
Snippet text here...
```

For direct LLM consumption. ~20 LOC.
